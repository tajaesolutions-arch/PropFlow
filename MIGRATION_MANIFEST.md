The repository contains two `202605060004_*` migrations; filename sorting applies `202605060004_billing_foundation.sql` before `202605060004_inventory_foundation.sql`.

Do not edit old migrations destructively after they have been applied to a shared or production Supabase project. Add a new forward-only migration for future production fixes.

## Runtime validation notes

- Apply all files in order before testing Vercel against the project.
- Refresh the Supabase/PostgREST schema cache after applying migrations that create or replace functions, especially workspace creation, direct-booking RPCs, iCal RPC-dependent views, and platform admin RPCs.
- The storage bucket must be named exactly `workspace-files` and must be private.
- PropFlow Admin is platform-only; never add `propflow_admin` to customer `workspace_members.roles`.
